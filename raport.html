Raport Scholarly HTML

1. Introducere

1.1 Scop

Aceasta aplicatie web este destinata elevilor de liceu si isi propune sa faciliteze invatarea programarii prin intermediul rezolvarii de probleme informatice. Problemele vor fi propuse de profesori si aprobate de administratori. Fiecare problema poate fi etichetata cu anumite TAG-uri, cum ar fi nivelul de dificultate (usoara/mediu/greu) sau capitolul informatic din care face parte (ex. programare dinamica, backtracking, etc.).

1.2 Conventii ale Documentului

Acest document respecta standardele IEEE si include informatii relevante despre functionalitatile esentiale ale aplicatiei, interactiunea cu utilizatorul si specificatiile tehnice.

1.3 Public Tinta si Sugestii de Citire

Publicul tinta al aplicatiei sunt elevii de liceu, profesorii si administratorii. Pentru o intelegere completa a documentului, se recomanda citirea sectiunilor in ordinea prezentata pentru fiecare tip de utilizator.

1.4 Scopul Produsului

Aplicatia este destinata sa faciliteze invatarea programarii pentru elevii de liceu, oferind o platforma interactiva pentru rezolvarea de probleme informatice. Aceasta este in concordanta cu obiectivele scolare si contribuie la dezvoltarea abilitatilor practice in domeniul informaticii.

2. Descrierea Generala

2.1 Perspectiva Produsului

Aplicatia web este un mediu virtual de invatare si rezolvare a problemelor informatice pentru elevii de liceu. Ea este conceputa sa ofere un cadru structurat pentru predare si invatare a programarii si sa faciliteze interactiunea intre elevi si profesori.

2.2 Functionalitati ale Produsului

Principalele functionalitati ale aplicatiei includ:

  • Propunerea si aprobarea de probleme de catre profesori si administratori
  • Crearea de clase si adaugarea de elevi in clase de catre profesori
  • Rezolvarea de probleme si trimiterea solutiilor pentru evaluare
  • Comentarea si evaluarea problemelor de catre elevi Exportarea si importarea problemelor intr-un format deschis
  • Generarea de rapoarte statistice pentru utilizatori si pentru fiecare problema

2.3 Clase si Caracteristici ale Utilizatorilor

Utilizatorii aplicatiei sunt impartiti in urmatoarele clase:

  • Elevi: rezolva probleme, trimit solutii pentru evaluare, pot comenta si evalua problemele
  • Profesori: propun si aproba probleme, creeaza si administreaza clase

2.4 Mediu de Operare

Aplicatia web va functiona pe orice dispozitiv cu acces la internet si un browser web modern. Este independenta de sistemul de operare si nu necesita instalare suplimentara de software.

2.5 Constraingeri de Design si Implementare

  • Utilizarea unui API REST/GraphQL pentru accesul la probleme si solutii
  • Implementarea unei interfete intuitive si usor de utilizat pentru toti utilizatorii

2.6 Documentatie pentru Utilizatori

Documentatia pentru utilizatori va include ghiduri de utilizare, tutoriale si documentatie online.

2.7 Presupuneri si Dependinte

  • Utilizarea unui sistem de autentificare sigura pentru accesul utilizatorilor
  • Disponibilitatea unei conexiuni la internet pentru utilizarea aplicatiei

3. Cerinte de Interfata Externa

3.1 Interfete cu Utilizatorii

Interfata cu utilizatorii va fi intuitiva si usor de navigat, oferind urmatoarele caracteristici:

  • Ecran de autentificare pentru accesul utilizatorilor
  • Interfata de navigare pentru accesul la clase, probleme si solutii
  • Formular de trimitere a solutiilor pentru evaluare
  • Sectiuni de comentarii si evaluare pentru fiecare problema

3.2 Interfete cu Hardware

Aplicatia web va comunica cu hardware-ul dispozitivelor utilizatorilor doar pentru afisarea informatiilor si trimiterea de date catre server.

3.3 Interfete cu Software-ul

Aplicatia va interactiona cu alte componente software, inclusiv:

  • Baza de date pentru stocarea problemelor, solutiilor si a altor date relevante
  • Servicii de autentificare si autorizare pentru gestionarea accesului utilizatorilor
  • API-uri externe pentru accesul la alte aplicatii web sau servicii

3.4 Interfete de Comunicare

Aplicatia va utiliza protocoale standard de comunicare, inclusiv HTTP/HTTPS, pentru interactiunea cu serverele si alte componente software.

4. Caracteristici ale Sistemului

4.1 Gestionarea Problemelor

Profesorii pot propune si aproba problemele, iar elevii le pot rezolva si trimite solutiile pentru evaluare.

4.1.1 Descriere si Prioritate

Profesorii pot asigna tag-uri problemelor si le pot marca cu o anumita prioritate (Usoara, Medie, Greu).

4.1.2 Secvente Stimul/Reactie

Profesorii propun problemele, elevii le rezolva si trimit solutiile pentru evaluare.

4.1.3 Cerinte Functionale

  • Profesorii pot crea probleme noi.
  • Elevii pot accesa problemele si le pot rezolva online.

4.2 Gestionarea Claselor

Profesorii pot crea si administra clasele, adaugand elevi si asignandu-le probleme.

4.2.1 Descriere si Prioritate

Crearea si administrarea claselor este o functionalitate de medie prioritate, deoarece este esentiala pentru organizarea si gestionarea elevilor.

4.2.2 Secvente Stimul/Reactie

Profesorii pot crea clase, adauga elevi si asigna probleme claselor.

4.2.3 Cerinte Functionale

  • Profesorii pot crea clase noi si le pot administra.
  • Profesorii pot adauga elevi in clase si le pot asigna probleme.

5. Alte Cerinte

5.1 Cerinte de Performanta

Aplicatia trebuie sa ofere o experienta rapida si fluida pentru utilizatori, indiferent de numarul de utilizatori simultani si de volumul de date procesate. Rata de incarcare a paginilor si timpul de raspuns al serverului trebuie sa fie optimi.

5.2 Cerinte de Siguranta

Aplicatia trebuie sa asigure protectia datelor utilizatorilor si sa previna accesul neautorizat la informatii sensibile. Toate datele trebuie sa fie stocate si transferate intr-un mod securizat conform standardelor de securitate.

5.3 Cerinte de Securitate

Aplicatia trebuie sa ofere autentificare sigura pentru utilizatori si sa implementeze masuri de securitate pentru protejarea datelor personale. De asemenea, trebuie sa aiba functionalitati de gestionare a accesului, asigurand ca fiecare utilizator are acces doar la resursele relevante pentru rolul sau.

5.4 Atribute de Calitate a Software-ului

Aplicatia trebuie sa fie adaptabila la schimbarile ulterioare si sa ofere o disponibilitate ridicata. De asemenea, trebuie sa fie corecta, flexibila, interoperabila si usor de intretinut. Fiabilitatea, reutilizabilitatea si testabilitatea trebuie sa fie prioritare pentru a asigura un produs de inalta calitate.

5.5 Reguli de Afacere

Aplicatia trebuie sa respecte regulile de afaceri specifice, cum ar fi permisiunile de acces pentru diferite roluri (profesor, elev, administrator) si alte restrictii legate de manipularea datelor si functionalitatile disponibile pentru fiecare utilizator.
